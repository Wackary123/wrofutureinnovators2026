🧠 FULL PROJECT SUMMARY — WRO FUTURE INNOVATORS (ATLAS)
🎯 Project Overview
This project is the development of ATLAS, an AI-powered robotic system designed to function as both a cultural museum guide and a smart security assistant. The goal is to enhance museum experiences by making them more interactive, educational, and secure, while also demonstrating how artificial intelligence can be applied in real-world environments.
ATLAS is built using a combination of:
Artificial Intelligence (OpenAI + Google Gemini)
Speech Recognition (Whisper)
Text-to-Speech (OpenAI TTS / optional alternatives)
Computer Vision (OpenCV)
Embedded hardware (Raspberry Pi, upgrading from Pi 3 → Pi 5)
Python-based software architecture
The system is designed to interact naturally with users through voice, recognize objects visually, and monitor surroundings for safety.

🤖 Core Functionalities
1. 🎤 AI Museum Guide (Primary Feature)
ATLAS acts as a conversational AI guide that:
Listens to user questions via microphone
Converts speech → text using Whisper
Sends text to an AI model (Gemini) for understanding and response
Generates answers based on cultural, historical, or exhibit-related knowledge
Speaks responses back using text-to-speech
Key Capabilities:
Natural conversation with users
Adaptive responses (based on language + context)
Explains exhibits in simple or advanced ways
Can ask follow-up questions to engage users

2. 📷 Computer Vision System
ATLAS uses a camera to:
Capture real-time images of surroundings
Detect basic visual features such as:
Colors
Shapes
Map detected features to known objects (e.g., paintings)
Example:
Detects a “red circle” → maps to “The Red Circle”
Automatically explains the detected artwork
This creates a hands-free, automatic tour guide experience.

3. 🚨 Security System (Secondary Feature — HIGH IMPACT)
ATLAS also functions as a smart security assistant.
Core Security Features:
Real-time environment monitoring using camera
Detection of unusual or suspicious activity
Artifact protection (prevent theft or damage)
Increased safety awareness for visitors and staff

🔐 Advanced Security Concept (Key Innovation)
A major focus of the project is addressing flaws in traditional security systems:
Problem:
Many systems rely on cloud-based storage
This makes them vulnerable to:
hacking
data breaches
system downtime
Real-world incidents show even large institutions can be compromised quickly
ATLAS Solution:
Uses local data storage (edge computing approach)
Processes and stores data directly on the device
Benefits:
Much harder to hack
No reliance on internet connection
Faster response time
Increased system reliability
Privacy Advantage:
User interactions are NOT sent to external servers
Protects visitor privacy
Important for environments where people may feel monitored
Inspiration:
Inspired by modern trends in local/edge computing systems
Similar to how companies are shifting away from centralized cloud-only models

4. 🔊 Audio System Debugging (IMPORTANT)
📌 Sound Device Identified
From ALSA:
card 2: snd_rpi_googlevoicehat_soundcar

❗ Issues Encountered
1. No sound initially
Audio routed to HDMI instead of HAT
Fixed by forcing:
aplay -D plughw:2,0 ...

2. Extremely low volume
Even at:
Playback Volume = 255 (max)
Sound was:
barely audible
only audible close to ear

🔬 Root Cause (CRITICAL INSIGHT)
From hardware capabilities:
FORMAT: S32_LE
CHANNELS: 2
RATE: 48000
AND:
👉 The Fusion HAT+ uses a tiny onboard mono speaker
 👉 Amplification is very weak by design
 👉 No separate gain control beyond digital volume

✅ Final Conclusion on Audio
✔️ System is functioning correctly
✔️ Audio pipeline is correct
❌ Speaker is physically too weak

🔥 Workaround / Solution
Software amplification using sox:
sox input.wav -d gain 20
Integrated into Python:
["sox", ..., "gain", "20"]
